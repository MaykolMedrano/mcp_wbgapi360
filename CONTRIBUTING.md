# Contributing to wbgapi360

Thank you for your interest in improving `wbgapi360`! We welcome contributions from economists, data scientists, and developers.

## How to Contribute

### 1. Reporting Bugs
- Check the [Issues](https://github.com/MaykolMedrano/mcp_wbgapi360/issues) tab to see if your issue has already been reported.
- If not, open a new issue using the "Bug Report" template.
- Include a minimal reproducible example (code snippet).

### 2. Suggesting Features
- Use the "Feature Request" template.
- Explain the economic or technical use case clearly.

### 3. Pull Requests
1.  **Fork** the repository.
2.  Create a new branch: `git checkout -b feature/my-new-feature`.
3.  Install dev dependencies: `pip install -e .[dev]`.
4.  Make your changes.
5.  **Run Tests**: Ensure `pytest tests/` passes.
6.  Commit your changes: `git commit -m 'Add some feature'`.
7.  Push to the branch: `git push origin feature/my-new-feature`.
8.  Open a **Pull Request**.

## Code Style
- Follow PEP 8 standards.
- Use type hints for all public functions.
- Add docstrings (Google style) for new methods.
